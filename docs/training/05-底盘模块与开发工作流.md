# 05. 底盘模块与开发工作流

## `crt_wheelchassis` 和 `uart_wheelchassis` 的区别

当前 `User/Module/Chassis/` 下面有两组和底盘相关的内容：

- `crt_wheelchassis/`
- `uart_wheelchassis/`

它们不是重复的，而是层次不同。

## `crt_wheelchassis` 是什么

`crt_wheelchassis` 是 **底盘基础控制框架**。

它主要负责：

- 底盘内电机对象
- 电机 PID
- 底盘运动学
- 目标速度接口
- 1ms 控制周期接口

**它不负责上位机速度输入、斜坡规划和串口应用层逻辑。**

### `crt_wheelchassis::Init()`

这个 `Init()` 主要初始化的是：

- **底盘内电机对象**
- **每个电机的 PID**
- **底盘基础控制结构**

## `uart_wheelchassis` 是什么

`uart_wheelchassis` 是基于 `crt_wheelchassis` 继续封装出来的一个 **串口控制应用示例**。

它新增了两类更接近真实应用的逻辑：

- 串口或上位机输入目标速度
- 用 `Class_Slope` 做速度平滑规划

### `Class_Uart_Wheelchassis::Init(CAN_HandleTypeDef* hcan)`

这个 `Init(...)` 里初始化的主要内容是：

- **复用底层底盘类的基础初始化**
- **重新绑定 4 个底盘电机使用的 CAN 总线**
- **初始化 `slope_vx_ / slope_vy_ / slope_vw_` 三个斜坡规划器**
- **把初始目标速度清零**

### `SetMotion(vx, vy, vw)`

作用：

- 保存上位机给定的原始目标速度
- 先做死区处理，避免小输入导致抖动

### `WithSlopeTask(vx, vy, vw)` / `WithSlopeTask()`

作用：

1. 更新目标速度
2. 让 `Class_Slope` 做平滑规划
3. 把规划结果写给底盘目标
4. 调用底盘 1ms 控制周期

## 如果我要写一个底盘控制逻辑，建议怎么做

### 第一步：算法层准备控制方法

常用的是：

- `alg_pid`
- `alg_filter`
- `alg_slope`

### 第二步：设备层准备电机对象

例如：

- `dvc_motor`
- `dvc_motor_dm`

### 第三步：模块层组织功能

如果你只是做底盘基础控制，先看：

- `crt_wheelchassis`

如果你要看“带速度规划的应用示例”，直接看：

- `uart_wheelchassis`

### 第四步：任务层做周期调度

任务层负责：

- 初始化顺序
- 回调分发
- 1ms 周期运行

不要把所有逻辑都直接塞进中断回调里。

## 新人最容易踩的几个坑

### 1. 只初始化电机对象，不初始化 CAN

如果只写：

```cpp
motor.Init(...);
```

但没有先写：

```cpp
CAN_Init(&hcan1, CAN_Motor_Call_Back);
```

那电机对象虽然存在，但总线和回调都没有准备好。

### 2. 只启动 CAN，不做回调分发

如果没有在 `tsk_config_and_callback.cpp` 里按 `StdId` 分发到对应设备对象，任务虽然在跑，但反馈不会更新。

### 3. 只算 PID，不发总线

很多新同学会调：

```cpp
motor.TIM_PID_PeriodElapsedCallback();
```

然后忘了：

```cpp
TIM_CAN_PeriodElapsedCallback();
```

这时控制量只是写进缓存，**并没有真正发出去。**

### 4. 忘了 `CAN2` 发送要自己加

当前示例代码里已经写好的发送对象是：

- **`CAN1`**

如果你后续需要 `CAN2`，必须自己去 `TIM_CAN_PeriodElapsedCallback()` 里补发送语句。

## 最后建议

第一次接触这个模板时，建议先把这三条主线打通：

1. **任务启动主线**：`freertos.c -> Task_Init() -> *_task`
2. **CAN 收发主线**：`CAN_Init() -> 回调分发 -> 设备对象 -> TIM_CAN_PeriodElapsedCallback()`
3. **底盘控制主线**：`uart_wheelchassis -> slope -> crt_wheelchassis -> motor`

把这三条主线看通了，后面再加传感器、加新电机、加新协议会轻松很多。
