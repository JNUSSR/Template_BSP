# 01. 工程总览与阅读顺序

## 这个工程是干什么的

这个工程是 **RoboCon 比赛用固件模板**，运行平台是 `STM32F427XX`，操作系统是 `FreeRTOS`，构建方式是 `CMake`。

模板已经准备好的内容包括：

- `STM32` 底层外设初始化框架
- `FreeRTOS` 任务调度框架
- `User/` 下的分层代码结构
- 电机、CAN、UART、TIM 等常用功能封装

**以后你平时主要改的地方就是 `User/`。**

## 根目录里最重要的文件和目录

### `Core/`

`CubeMX` 生成的核心代码目录，里面有：

- `main.c`
- `freertos.c`
- 外设初始化代码
- 中断处理代码

**程序怎么启动，先看这里。**

### `Drivers/`

`STM32 HAL`、`CMSIS` 等官方库目录。平时业务开发一般不改。

### `Middlewares/`

中间件目录，这个工程里最重要的是 `FreeRTOS` 源码。一般只使用，不直接修改。

### `User/`

战队自己的核心代码目录。算法、驱动、设备、模块、任务基本都在这里。

### `CMakeLists.txt`

工程总构建文件。

当前模板已经做了这两件事：

- **自动扫描 `User/` 下所有 `.c/.cpp` 文件**
- **自动收集 `User/` 下头文件所在目录**

所以结论可以直接记：

- **只要新文件放在 `User/` 里，新增文件时不需要改 `CMakeLists.txt`。**

但新增文件后仍然建议：

1. 重新 `Configure CMake`
2. 编译一次
3. 确认没有漏头文件、漏路径

## 第一次读工程，建议按这个顺序

1. **先看 [freertos.c](F:/Code/ROBOCON/2026ROBOCON/Task/BSP+FreeRTOS/Template_Cmake/Core/Src/freertos.c)**
2. 再看 `User/Task/tsk_config_and_callback/`
3. 再看某个 `*_task`
4. 再看对应的 `dvc_*`
5. 最后再看 `alg_*`

## 读代码的方法

读某个模块时，建议固定按这个顺序：

1. **先看 `.h` 头文件**
2. **先看类定义、函数声明、结构体、枚举、成员变量**
3. **先搞清楚这个模块“对外提供什么接口”**
4. 再去看 `.cpp` 里的实现细节

原因很简单：

- **头文件决定“这个模块有什么”**
- **源文件决定“这个模块怎么做”**

所以不要一上来就扎进 `.cpp` 看实现，**先把 `.h` 看明白，效率更高。**
