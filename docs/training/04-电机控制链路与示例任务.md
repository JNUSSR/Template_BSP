# 04. 电机控制链路与示例任务

## 这两个任务是什么性质

当前 `User/Task/` 下面的两组电机任务：

- `Motor_DJI`
- `Motor_DM`

它们都是 **示例任务**。

具体来说：

- `Motor_DJI` 是 **DJI3508 示例任务**
- `Motor_DM` 是 **DM4310 示例任务**

它们的主要作用是演示：

- 怎样初始化电机
- 怎样绑定 CAN、ID、控制模式
- 怎样在任务里跑周期控制
- 怎样接收反馈并更新状态

## 大疆电机对象的关键调用

当前模板里主要看 `Class_Motor_C620`。

### `Init(...)`

作用：

- 绑定使用哪一路 CAN
- 指定电机 ID
- 指定控制模式
- 设置减速比、最大扭矩
- 建立发送缓存映射

**`Init(...)` 初始化的主要内容是：**

- **电机使用哪一路 CAN 总线**
- **电机的 CAN ID**
- **控制模式**
- **减速比**
- **最大扭矩**
- **内部发送缓存映射**

典型调用：

```cpp
motor_3508.Init(&hcan1, CAN_Motor_ID_0x201, Control_Method_OMEGA);
```

含义是：

- 这台电机走 `CAN1`
- 电机 ID 是 `0x201`
- 控制方式是速度模式

### `CAN_RxCpltCallback(...)`

作用：

- 接收一帧电机反馈
- 解析编码器、速度、扭矩、温度
- 更新 `Now_Angle`、`Now_Omega`、`Now_Torque`

### `TIM_PID_PeriodElapsedCallback()`

作用：

1. 读取目标值
2. 读取当前反馈值
3. 执行 PID 计算
4. 把输出写进对应 CAN 发送缓存

注意：

- **它负责算，不负责真正发总线**
- **真正发总线靠 `TIM_CAN_PeriodElapsedCallback()`**

## DJI3508 示例任务怎么初始化

在 [motor_dji_task.cpp](F:/Code/ROBOCON/2026ROBOCON/Task/BSP+FreeRTOS/Template_Cmake/User/Task/Motor_DJI/motor_dji_task.cpp) 中，关键初始化是：

```cpp
motor_3508.PID_Omega.Init(450.0f, 279.936f, 0.0f, 0.0f, 10000.0f, 10000.0f);
motor_3508.Init(&hcan1, CAN_Motor_ID_0x201, Control_Method_OMEGA);
```

这里要拆成两部分理解：

### 1. PID 初始化

`motor_3508.PID_Omega.Init(...)`

作用：

- 准备速度环参数

### 2. 电机对象初始化

`motor_3508.Init(...)`

作用：

- 准备 CAN 总线
- 准备电机 ID
- 准备控制模式
- 准备发送缓存绑定

## 从上电到闭环控制的主链路

下面按当前模板的真实代码顺序走一遍。

### 第一步：底层外设初始化

在 `main.c` 和 `CubeMX` 生成代码中，先完成：

- 时钟初始化
- GPIO 初始化
- CAN 初始化
- UART 初始化

### 第二步：进入 `MX_FREERTOS_Init()`

在 `freertos.c` 里先调用：

```c
Task_Init();
```

### 第三步：`Task_Init()` 统一初始化

这里会做：

1. `BSP_Init(...)`
2. `CAN_Init(&hcan1, CAN_Motor_Call_Back)`
3. `DJI_M3508_Task_Init()`
4. `J4310_Task_Init()`

### 第四步：电机示例任务初始化

以 `DJI3508` 为例：

1. 初始化速度环 PID
2. 初始化电机对象

### 第五步：线程开始运行

`DJI_M3508_Task()` 循环里当前做的是：

1. `Set_Target_Omega(...)`
2. `TIM_PID_PeriodElapsedCallback()`
3. `TIM_CAN_PeriodElapsedCallback()`
4. `osDelay(1)`

### 第六步：发送控制命令

`TIM_CAN_PeriodElapsedCallback()` 会把缓存里的控制量真正发出去。

这里要特别注意：

- **当前示例代码发的是 `CAN1`**
- **如果你的设备挂在 `CAN2`，要自己补 `CAN_Send_Data(&hcan2, ...)`**

### 第七步：接收电机反馈

电机反馈回来后，会经过这条链：

```text
HAL_CAN_RxFifo0MsgPendingCallback()
  -> CAN1_Manage_Object.Callback_Function(...)
  -> CAN_Motor_Call_Back(...)
  -> DJI_M3508_CAN_RxCpltCallback(...)
  -> motor_3508.CAN_RxCpltCallback(...)
```

### 第八步：设备对象更新状态

`motor_3508.CAN_RxCpltCallback(...)` 会更新：

- `Now_Angle`
- `Now_Omega`
- `Now_Torque`

这样下一轮 `PID` 才能继续闭环。

## 一句话总结

完整链路可以记成：

```text
初始化 CAN
-> 注册 CAN 回调
-> 初始化电机 PID
-> 初始化电机对象
-> 任务循环设置目标值
-> 电机对象执行 PID 计算
-> CAN 发送函数周期下发
-> 电机返回反馈
-> 回调分发给电机对象
-> 电机对象更新反馈状态
-> 下一轮继续闭环
```
